# 🌍 Hevan Booking - نظام إدارة الحجوزات السياحية

**بسم الله الرحمن الرحيم**

---

## 1. شنو المشروع ده؟

**Hevan Booking** هو نظام ويب متكامل لإدارة الحجوزات السياحية، معمول عشان يساعد الشركات السياحية في السودان وفي أي بلد تاني عشان تدير حجوزاتها إلكتروني. المشروع ده مبني على بحث أكاديمي بعنوان **"نظام إدارة الحجوزات السياحية"** وبيحول الفكرة النظرية لتطبيق عملي جاهز للاستخدام.

المشروع بيهدف يحل مشاكل:
- **الأنظمة اليدوية** في الحجوزات (ورقة وقلم)
- **ضعف الترويج** للسياحة السودانية
- **صعوبة التواصل** بين الشركات والسائحين
- **ضياع البيانات** وتكرارها

## 2. التقنيات المستخدمة

| التقنية | الاستخدام |
|---------|-----------|

| **Docker** | بيئة تشغيل بديلة (PHP 8.2-Apache) |
| **SVG Icons** | أيقونات SVG داخلية - بدون مكتبات خارجية |

## 3. كيفية التشغيل على XAMPP (خطوة بخطوة)

### الخطوة 1: نزّل XAMPP
لو ما عندك XAMPP، نزّله من الموقع الرسمي: https://www.apachefriends.org

### الخطوة 2: انسخ ملفات المشروع
انسخ **محتوى مجلد `dev/`** كله إلى:
```
C:\xampp\htdocs\hevan-booking
```
يعني تكون الملفات جوا المجلد كده:
```
C:\xampp\htdocs\hevan-booking\
    ├── index.php
    ├── config.php
    ├── database.sql
    ├── assets/
    ├── admin/
    ├── includes/
    └── ... باقي الملفات
```

### الخطوة 3: شغّل الخدمات
افتح **XAMPP Control Panel** و shut up (شغّل) الخدمات دي:
- **Apache** (الزر بتاع Start)
- **MySQL** (الزر بتاع Start)

### الخطوة 4: أنشئ قاعدة البيانات
افتح المتصفح وادخل على:
```
http://localhost/phpmyadmin
```
- اضغط على **New** (جديد)
- اكتب اسم القاعدة: `hevan_booking`
- اختار `utf8mb4_unicode_ci` في الكولايشن
- اضغط **Create**

### الخطوة 5: استورد قاعدة البيانات
- من phpMyAdmin، اختار قاعدة `hevan_booking`
- اضغط على تبويب **Import** (استيراد)
- اختار ملف `database.sql` من مجلد المشروع
- اضغط **Go**

الملف ده بيحتوي على:
- جداول النظام (users, companies, places, bookings)
- بيانات تجريبية (شركات، مناطق، حجوزات)
- مستخدم إداري (admin / admin123)

### الخطوة 6: افتح المشروع
في المتصفح اكتب:
```
http://localhost/hevan-booking
```

وبكده يفتح معاك الموقع والشغال تمام!

## 4. كيفية التشغيل على Docker

إذا كنت بتستخدم Docker بدل XAMPP، فيه طريقة أسهل:

### باستخدام السكريبت المرفق
من جذر الريبو (مش جوا `dev/`)، شغّل الأمر ده في الطرفية:
```bash
./refresh-docker-web.sh
```

### يدوي
```bash
# في نفس مكان ملف docker-compose.yml
docker compose up -d --build
```

وبعدها افتح المتصفح على:
```
http://127.0.0.1:8080
```

### إيه الفرق؟
في Docker، النظام بيشتغل على صورة `php:8.2-apache` مع إضافة MySQL في container منفصل. الإعدادات بتتنقل عبر environment variables:
- `DB_HOST=db` (اسم service الـ MySQL)
- `DB_USER=hevan`
- `DB_PASS=hevanpass`
- `DB_NAME=hevan_booking`

## 6. شرح قاعدة البيانات والجداول

النظام يستخدم 4 جداول أساسية:

### 6.1 جدول users (المستخدمين)
```sql
- id: رقم المستخدم (تلقائي)
- full_name: الاسم الكامل
- username: اسم المستخدم (فريد)
- password: كلمة المرور (مشفرة bcrypt)
- email: البريد الإلكتروني
- role: الصلاحية (admin / manager)
- created_at: تاريخ الإنشاء
- updated_at: تاريخ التعديل
```
**المستخدم الافتراضي:** admin / admin123

### 6.2 جدول companies (الشركات السياحية)
```sql
- id: رقم الشركة
- name: اسم الشركة
- description: وصف الشركة
- email: البريد الإلكتروني
- phone: رقم الهاتف
- address: العنوان
- logo: مسار الصورة
- is_active: مفعلة ولا لأ (1/0)
- created_at / updated_at
```
**البيانات التجريبية:** 5 شركات (سياحة القدس، شواطئ غزة، جبال نابلس، أريحا للسياحة العلاجية، وادي الأردن)

### 6.3 جدول places (المناطق السياحية)
```sql
- id: رقم المنطقة
- company_id: رقم الشركة (ربط مع جدول companies)
- name: اسم المنطقة
- description: وصف المنطقة
- location: الموقع التقريبي
- city: المدينة
- image: مسار الصورة
- price_per_night: السعر لليلة
- max_guests: أقصى عدد ضيوف
- category: التصنيف (فندق، شقة، شاليه، مزرعة، إلخ)
- rating: التقييم
- is_featured: مميزة ولا لأ
- is_active: مفعلة ولا لأ
```
**البيانات التجريبية:** 12 وجهة سياحية متنوعة

### 6.4 جدول bookings (الحجوزات)
```sql
- id: رقم الحجز
- place_id: رقم المنطقة (ربط مع places)
- customer_name: اسم العميل
- customer_email: البريد
- customer_phone: رقم الهاتف
- check_in: تاريخ الوصول
- check_out: تاريخ المغادرة
- guests: عدد الضيوف
- total_price: السعر الإجمالي
- status: الحالة (pending / accepted / rejected / cancelled)
- notes: ملاحظات
```
**الحالات المدعومة:** قيد الانتظار (pending)، مقبول (accepted)، مرفوض (rejected)، ملغي (cancelled)

## 7. كيفية الدخول (بيانات الإداري)

### الصفحة العامة
ما في داعي تسجل دخول - أي زائر يقدر يشوف المناطق والشركات ويسجل حجز.

### صفحة الإداري (لوحة التحكم)
عشان تدخل لوحة التحكم، روح لـ:
```
http://localhost/hevan-booking/login.php
```
واستخدم البيانات دي:

| الحقل | القيمة |
|-------|--------|
| **اسم المستخدم** | `admin` |
| **كلمة المرور** | `admin123` |

بعد ما تسجل دخول، بتظهر ليك لوحة التحكم وفيها:
- ✅ إحصائيات سريعة (عدد الشركات، المناطق، الحجوزات)
- ✅ جدول بآخر الحجوزات
- ✅ إدارة الشركات (إضافة/تعديل/حذف)
- ✅ إدارة المناطق السياحية (إضافة/تعديل/حذف)
- ✅ إدارة الحجوزات (تغيير الحالة/حذف)

## 8. مميزات المشروع

### 🎯 للمستخدم العادي (الزائر):
- **عرض المناطق السياحية** مع صور من assets (تظهر تلقائياً حسب اسم المنطقة)
- **عرض الشركات السياحية** مع معلومات الاتصال
- **تسجيل حجز جديد** بسهولة - فقط املأ النموذج
- **بحث سريع** عن وجهة أو شركة
- **وضع ليلي/نهاري** (Dark/Light Mode) - ينقلب تلقائي
- **تصميم متجاوب** - يشتغل على الجوال واللابتوب والتابلت

### 🛠️ للإداري (admin):
- **لوحة تحكم** بإحصائيات مباشرة
- **إدارة الشركات** - إضافة شركة جديدة، تعديل بياناتها، حذفها
- **إدارة المناطق** - إضافة وجهة سياحية، تعديل وصفها، حذفها
- **إدارة الحجوزات** - قبول أو رفض الحجز، فلترة حسب الحالة، حذف
- **تحديث الحالة** بنقرة زر واحدة

### 🖼️ الصور الذكية:
النظام بيعرف يربط أسماء المناطق بالصور الموجودة في `assets/` تلقائياً:
- منطقة فيها كلمة "مروي" ← تظهر صور `Meroë-Pyramids1.png` و `2` و `3` و `4`
- منطقة فيها كلمة "توتي" ← تظهر صور `Tuti-Island1.png` و `2` و `3` و `4`
- منطقة فيها كلمة "سواكن" ← تظهر صور `Suakin1.png` و `2` و `3` و `4` و `5`
- منطقة فيها كلمة "جبل مرة" ← تظهر صور `Jabal-Marra1.png` و `2` و `3` و `4`
- منطقة فيها كلمة "كسلا" أو "شلال" ← تظهر صورة `Totel-Mountain-Waterfalls1.png`
- أي منطقة تانية ← تظهر الصورة الافتراضية `hram.png`

### 🌐 SVG Icons داخلية:
كل الأيقونات في الموقع معمولة بـ SVG خالص - ما في أي مكتبة خارجية (Font Awesome ولا Bootstrap Icons). الأيقونات تشمل: globe, booking, landscape, lock, search, building, pin, mobile, user, phone, mail, calendar, note, save, back, key, bulb, chart, trash, edit, plus, clock, check, x, sun, moon.

### 🎨 الدارك مود (Dark/Light Mode):
الموقع يدعم التبديل بين الوضع الفاتح والداكن. اختيارك بينحفظ في localStorage ويرجع تاني حتى لو سويت Refresh.

## 10. troubleshooting (مشاكل وحلول)

| المشكلة | السبب | الحل |
|---------|-------|------|
| خطأ اتصال قاعدة البيانات | MySQL مش شغال أو القاعدة ما اتضافت | شغّل MySQL في XAMPP وتأكد إن `hevan_booking` موجودة |
| الصفحة فاضية (404) | المسار غلط | تأكد إن المجلد اسمه `hevan-booking` جوا `htdocs` |
| الصور ما بتظهر | الصورة مش موجودة في assets | حط الصورة في مجلد `assets/` أو غير اسمها |
| الدخول ما يشتغل | كلمة السر غلط | استخدم admin / admin123 أو أعد استيراد `database.sql` |
| Docker ما يشتغل | الكونفلكت في البورت | غير البورت في docker-compose.yml |
