## 1. مقدمة (Introduction)

هذا المستند يحدد المتطلبات الوظيفية وغير الوظيفية لنظام **إدارة الحجوزات السياحية**،
وهو تطبيق ويب PHP/MySQL يمكّن الزوار من استعراض الشركات والمناطق السياحية وتسجيل الحجوزات،
ويمنح الإداري صلاحية إدارة المحتوى والحجوزات بالكامل.

## 3. المتطلبات غير الوظيفية (Non-Functional Requirements)

| المعرف | المتطلب | الوصف | المعيار |
|--------|---------|-------|---------|
| NFR-01 | **واجهة عربية RTL** | كل الواجهات من اليمين لليسار، واللغة عربية | النصوص والتنسيقات كلها عربية |
| NFR-02 | **أداء سريع** | زمن تحميل الصفحات أقل من 3 ثواني | استعلامات SQL محسّنة |
| NFR-03 | **تصميم متجاوب (Responsive)** | يعمل على موبايل وتابلت ولا بتوب | CSS Media Queries |
| NFR-04 | **الأمان** | حماية صفحات الإداري من الوصول المباشر | التحقق من الجلسة في كل صفحة إدارية |
| NFR-05 | **خطأ مفهوم** | رسائل خطأ واضحة بالعربية عند فشل الاتصال أو التحقق | رسائل في alert boxes |
| NFR-06 | **قابلية التوسع** | إمكانية إضافة جداول ووظائف جديدة بسهولة | كود منظم، ملفات مفصولة |
| NFR-07 | **الحفظ التلقائي** | البيانات التجريبية تنشأ مع قاعدة البيانات | SQL seed data |

## 4. هيكل قاعدة البيانات (Database Schema)

### جدول: `users`
```sql
id           INT AUTO_INCREMENT PRIMARY KEY
name         VARCHAR(120)
username     VARCHAR(80) UNIQUE
email        VARCHAR(150) NULL
password     VARCHAR(255)
role         ENUM('admin','user')
created_at   TIMESTAMP
```

### جدول: `companies`
```sql
id           INT AUTO_INCREMENT PRIMARY KEY
name         VARCHAR(180)
address      VARCHAR(255)
phone        VARCHAR(60)
description  TEXT NULL
image_url    VARCHAR(255) NULL
created_at   TIMESTAMP
```

### جدول: `places`
```sql
id           INT AUTO_INCREMENT PRIMARY KEY
name         VARCHAR(180)
location     VARCHAR(180)
category     VARCHAR(100) DEFAULT 'عام'
description  TEXT NULL
image_url    VARCHAR(255) NULL
created_at   TIMESTAMP
```

### جدول: `bookings`
```sql
id           INT AUTO_INCREMENT PRIMARY KEY
user_name    VARCHAR(120)
phone        VARCHAR(60)
email        VARCHAR(150) NULL
company_id   INT (FK → companies.id)
place_id     INT (FK → places.id, ON DELETE SET NULL)
booking_date DATE
notes        TEXT NULL
status       ENUM('pending','accepted','rejected')
created_at   TIMESTAMP
```

## 7. Acceptance Criteria (معايير القبول)

| السيناريو | الخطوات | النتيجة المتوقعة |
|-----------|---------|-----------------|
| فتح الموقع | الدخول إلى `index.php` | ظهور الصفحة الرئيسية مع hero + إحصائيات + grid المناطق والشركات |
| حجز جديد | تعبئة النموذج في `book.php` | رسالة "تم تسجيل الحجز بنجاح" + الحفظ في قاعدة البيانات |
| دخول الإداري | إدخال admin/admin123 في `login.php` | التوجيه إلى لوحة التحكم مع الإحصائيات |
| إدارة الشركات | الدخول إلى `admin/companies.php` | إضافة/تعديل/حذف شركات، مع رسائل تأكيد |
| إدارة الحجوزات | الدخول إلى `admin/bookings.php` | تغيير الحالة pending/accepted/rejected وحذف |
| تأمين الصفحات | محاولة فتح صفحة إدارية بدون تسجيل | توجيه تلقائي إلى login.php |
