## 1. ملخص تنفيذي (Executive Summary)

مشروع **"نظام إدارة الحجوزات السياحية"** هو تطبيق ويب يهدف إلى رقمنة عمليات الشركات السياحية في السودان،
وتوفير منصة لعرض الوجهات السياحية، وإدارة الحجوزات إلكترونياً بدلاً من الأنظمة اليدوية الحالية.

النظام مبني على بحث أكاديمي متكامل ويقدم حلاً عملياً لمشاكل:
- غياب الأنظمة الإلكترونية للحجوزات
- ضعف الترويج للسياحة السودانية
- صعوبة التواصل بين الشركات السياحية والسائحين
- الفقدان والتكرار في البيانات بسبب العمل اليدوي

## 2. أهداف المشروع (Business Objectives)

| الهدف | الوصف | مؤشر النجاح |
|-------|-------|-------------|
| 🎯 رقمنة الحجوزات | استبدال العمليات اليدوية بنظام إلكتروني متكامل | 100% من الحجوزات عبر النظام |
| 📢 الترويج السياحي | توفير منصة لعرض الوجهات السياحية السودانية | تغطية 5+ ولايات سياحية |
| ⚡ تسريع الإجراءات | تقليل وقت الحجز من أيام إلى دقائق | حجز كامل في أقل من 5 دقائق |
| 📊 تحسين الإدارة | توفير لوحة تحكم لإدارة الشركات والحجوزات والمناطق | إدارة كاملة عبر واجهة واحدة |

---

## 3. نطاق المشروع (Scope)

### 3.1 داخل النطاق (In-Scope)
- ✅ عرض المناطق السياحية السودانية مع وصف وتصنيف
- ✅ عرض الشركات السياحية المسجلة
- ✅ تسجيل حجوزات جديدة من قبل الزوار
- ✅ لوحة تحكم إدارية (Admin Dashboard)
- ✅ إدارة الشركات السياحية (إضافة/تعديل/حذف)
- ✅ إدارة المناطق السياحية (إضافة/تعديل/حذف)
- ✅ إدارة الحجوزات (عرض/تحديث حالة/حذف)
- ✅ نظام مصادقة للإداري
- ✅ بيانات تجريبية لعرض المناقشة

### 3.2 خارج النطاق (Out-of-Scope) — للإصدارات المستقبلية
- ❌ نظام دفع إلكتروني
- ❌ حسابات مستخدمين متعددة مع صلاحيات مختلفة
- ❌ نظام تقييم وتعليقات
- ❌ تطبيق موبايل
- ❌ تكامل مع بوابات الدفع المحلية
- ❌ خرائط تفاعلية للمناطق السياحية
- ❌ دعم متعدد اللغات

## 4. المستخدمون المستهدفون (Target Users)

| المستخدم | الوصف | الاحتياجات الرئيسية |
|----------|-------|---------------------|
| 👤 **الزائر / السائح** | شخص يبحث عن شركات سياحية ومناطق سياحية في السودان | • تصفح المناطق السياحية<br>• تصفح الشركات<br>• تسجيل حجز جديد<br>• سهولة الاستخدام |
| 👨‍💼 **الإداري** | مدير النظام المسؤول عن إدارة المحتوى والحجوزات | • لوحة تحكم بإحصائيات<br>• إضافة/تعديل/حذف شركات<br>• إضافة/تعديل/حذف مناطق<br>• إدارة الحجوزات (قبول/رفض)<br>• متابعة الأداء |

## 5. قواعد العمل (Business Rules)

| الرمز | القاعدة |
|------|---------|
| BR-01 | لا يمكن تسجيل حجز بدون اختيار شركة سياحية |
| BR-02 | الحجز الجديد يكون بحالة "قيد الانتظار" افتراضياً |
| BR-03 | الإداري فقط يمكنه تغيير حالة الحجز (مقبول/مرفوض) |
| BR-04 | لا يمكن حذف شركة مرتبطة بحجوزات (حذف متتالي) |
| BR-05 | يمكن حذف منطقة دون حذف الحجوزات المرتبطة (Set Null) |
| BR-06 | اسم المستخدم ورقم الهاتف إجباريان للحجز |
| BR-07 | بيانات الدخول للإداري محددة مسبقاً (admin/admin123) |
| BR-08 | النظام يعرض فقط المحتوى النشط والمتاح |

## 6. تدفق العمل الأساسي (Core Workflow)

### 6.1 رحلة الزائر (User Journey)
```
الصفحة الرئيسية → تصفح المناطق ← اختيار منطقة
                                 ↓
                    تسجيل حجز (الاسم، الهاتف، الشركة، التاريخ)
                                 ↓
                    تأكيد الحجز (حالة: قيد الانتظار)
```

### 6.2 رحلة الإداري (Admin Journey)
```
تسجيل الدخول (admin/admin123)
              ↓
    ┌──────────────────────────────────┐
    │      لوحة التحكم (إحصائيات)       │
    ├──────────────────────────────────┤
    │  إدارة الشركات → إضافة/تعديل/حذف  │
    │  إدارة المناطق → إضافة/تعديل/حذف  │
    │  إدارة الحجوزات → قبول/رفض/حذف    │
    └──────────────────────────────────┘
```

## 7. التقنيات المقترحة (Technology Stack)

| الطبقة | التقنية | السبب |
|--------|---------|-------|
| **الواجهة الأمامية** | HTML5, CSS3, JavaScript | يعمل على أي متصفح، متوافق مع XAMPP |
| **Backend** | PHP 8+ | الأكثر انتشاراً مع XAMPP، سهل للطلبة |
| **قاعدة البيانات** | MySQL (MariaDB) | مجاني، متكامل مع phpMyAdmin |
| **الخادم** | Apache (XAMPP) | أسهل بيئة تشغيل على ويندوز |
| **العرض المباشر** | Ngrok / Cloudflare Tunnel | لعرض الديمو عبر الإنترنت |

---

## 8. المخاطر (Risks & Mitigation)

| الخطر | التأثير | خطة التعامل |
|-------|---------|-------------|
| XAMPP غير مثبت على جهاز الطالب | لا يمكن تشغيل المشروع | توفير تعليمات تثبيت XAMPP بالصور |
| مشاكل اتصال بقاعدة البيانات | النظام لا يعمل | رسائل خطأ واضحة بالعربية |
| الطلبة ما عندهم خبرة PHP كافية | صعوبة في التعديل | توثيق الكود عربي، شروحات لكل ملف |
| وقت المناقشة محدود | عرض غير كامل | بيانات افتراضية جاهزة، تركيز على الوظائف الرئيسية |
